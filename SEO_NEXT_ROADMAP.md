# SEO Next Roadmap

## 1. Переделать новые страницы под стиль LORA

Главный следующий этап — не технический, а продуктово-визуальный: пройти по каждой новой странице и спроектировать для нее отдельное дизайн-решение в стиле LORA.

Нужно проработать:

- `/services/`
- `/industries/`
- `/blog/`
- все страницы услуг;
- все страницы ниш;
- `/pricing/`
- `/cases/`
- `/contacts/`
- `/audit/`
- `/privacy/`
- `/cookies/`
- `/thank-you/`

Цель:

- сохранить весь SEO-фундамент;
- не ломать URL, meta, sitemap, JSON-LD, hreflang и аналитику;
- уйти от ощущения шаблонных SEO-страниц;
- сделать каждую страницу самостоятельной посадочной страницей LORA;
- доработать композицию, первый экран, CTA, текстовые блоки, визуальные акценты и мобильный вид.

Работать лучше постранично:

1. Выбрать страницу.
2. Определить ее роль: продажа услуги, нишевая посадочная, навигационная, служебная, контактная.
3. Спроектировать уникальную структуру.
4. Обновить компонент или конфиг.
5. Проверить desktop/mobile.
6. Проверить, что SEO-разметка осталась на месте.

## 2. Добавить prerender / SSG для всех sitemap routes

Сейчас проект остается Vite SPA. Route-specific meta и JSON-LD появляются после выполнения React.

Нужно сделать так, чтобы каждый URL из sitemap сразу отдавал свой HTML:

- уникальный `<title>`;
- уникальный description;
- canonical;
- hreflang;
- `x-default`;
- route-specific JSON-LD;
- `noindex` для языковых thank-you страниц.

Это главный технический SEO-апгрейд после визуальной переработки страниц.

## 3. Исправить robots для языковых thank-you URL

Сейчас `robots.txt` содержит:

```txt
Disallow: /thank-you/
```

Но реальные URL языковые:

- `/en/thank-you/`
- `/es/thank-you/`
- `/ru/thank-you/`
- `/zh/thank-you/`

Нужно добавить правила под языковые версии или wildcard, чтобы robots соответствовал реальной URL-структуре.

## 4. Доуникализировать контент услуг и ниш

Часть страниц уже получила уникальные problems, deliverables, integrations, FAQ и related links. Остальные еще частично используют общий базовый слой.

Нужно пройти все услуги и ниши:

- уникальный first-screen offer;
- уникальные проблемы;
- уникальные deliverables;
- уникальные integrations;
- уникальный FAQ;
- уникальные CTA;
- осмысленные related links;
- без фейковых кейсов, отзывов, цифр и гарантий.

## 5. Разбить JS bundle

Сейчас build проходит, но Vite предупреждает, что JS chunk больше 500 kB.

Нужно рассмотреть:

- `React.lazy` для SEO-страниц;
- lazy loading тяжелых секций главной;
- ручное разбиение chunks через Vite/Rollup;
- проверку, какие ассеты реально попадают в основной bundle.

## 6. Подключить реальные счетчики и вебмастеры

Analytics layer уже подготовлен, но реальные счетчики еще не подключены.

Нужно:

- подключить Яндекс Метрику;
- подключить Google Analytics, если нужен;
- подтвердить Google Search Console;
- подтвердить Яндекс Вебмастер;
- отправить sitemap;
- проверить цели по событиям из `analytics.ts`;
- проверить consent/cookie-логику перед полноценной рекламой.

## 7. Расширить блог и кейсы

Для дальнейшего organic SEO нужны реальные материалы:

- статьи под коммерческие и информационные запросы;
- страницы с реальными кейсами;
- разборы процессов;
- материалы под ниши;
- материалы под услуги.

Важно: не добавлять выдуманные результаты, клиентов, отзывы и гарантии.
